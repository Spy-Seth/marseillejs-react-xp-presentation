Enseignements tirés

❤︎ ❤︎ ❤︎ React ❤︎ ❤︎ ❤︎

  • React seul est extrêment simple
  • Le JSX est un vrai bonheur à utiliser
  • Flux est un pattern théorique inutilisable en l'état
  • Parvenir à l'isomorphisme est très contraignant
  • Stack complexe à mettre en œuvre : système de build à créer de zéro

NB : Depuis nos débuts avec cette stack beaucoup de progrès ont été réalisés. Il est maintenant beaucoup plus facile d'initialiser un projet React/Flux isomorphique.

Approfondissement

« React seul est extrêment simple » 1/2

React est un simple appel à une fonction avec des paramètres


React.render(
	<SayHello to="Jean-Michel" />,
	document.getElementById('example-hello-world')
);
					

React.render(
	React.createElement(sayHello, {to: "Jean-Michel"},
	document.getElementById('say-hello-container')
);

« React seul est extrêment simple » 2/2

  • Isolation parfaite des composants entre eux
  • Le JSX exprime l'état final du DOM
    • on se concentre sur le HTML que l'on veut générer...
    • ... et non sur la mutabilité du DOM
  • Rend explicite le métier de la GUI

Exemple


var DealListPage = React.createClass({
    /* ... */
    render: function () {
        var city = this.state.city;
        var now = Date.now();

        return (
            <div className="deal-list-page-component">
                {city.isSellOpen(now) ?
                    <OpenSellCity city={city} /*...*/ />
                    :
                    <ClosedSellCity city={city} />
                }
            </div>
        );
    }
});

« Parvenir à l'isomorphisme est très contraignant »

  • Il faut être capable de démarer l'application coté client à partir de l'état dans lequel on l'a laissé coté serveur
  • Transfert des données entre le client et le serveur
  • Partage des composants
  • Différences importantes entre les deux environements
